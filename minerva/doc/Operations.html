Maintenance Mode

The system can be put in maintenance mode by setting the parameter maintenanceMode in the .../WEB-INF/web.xml file to 1. While maintenance mode is enabled, Minerva will display a "system unavailable" message in place of the welcome message, the search bar is removed, and the menu is not donwloaded or shown. The top level navigation bar is still shown and active as some of those links can point to other systems and locations within the WHO website. Those links that point back to the Minerva client are active but will of course still result in the "system unavailable" message.

To disable maintenance mode and return to normal operations, simply set the maintenanceMode parameter to 0.

This function is turned off (set to 0) by default

Enabling wiki link

To enable the wiki linkage, set the wikiURL parameter in web.xml to the URL root of the dokuwiki that is used for the GHO. This will both set up the link and display the "Notes and discussion" tab on the interface.

This function is not set by default

Enabling Trac link

Minerva can be connected to a Trac ticket system so that tickets can be logged and tracked against individual data views. In web.xml, set the tracURL parameter to the location of your trac ticket system (dont put a trailing / character). For example, the link for the gho at WHO is

https://extranet.who.int/trac/GHODATA
You must also specify the trac username and password to be able to access the JSON web service that provides queries into the ticket DB. This is done using the parameters tracUsername and tracPassword.
The Trac ticket tracker MUST have the following custom fields defined: vid and theme. Refer to the Trac documentation if you do not know how to do this. You must also create a component called "DataVIew".

This function is disabled by default as it is not something we wished exposed to the general public. It should only be made available to the WHO community.
